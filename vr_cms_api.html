1.影院管理

1.1创建影院

request

POST /cms/cinemas

Content-Type : text/plain

{
  "cinemaCode": "huabeiqu_cinema_0001",
  "name": "X立方长宁缤谷",
  "regionID": "BEIJING",
  "description": "第二家店",
  "address": "北京市朝阳区测试道路1号",
  "longitude": 116.3742256165,
  "latitude": 39.8881383092,
  "phone": "010-66667777"
}
名称 类型 描述 是否必须
cinemaCode string 影院的唯一的识别码，可读 是
name string 影院名称，限制在256字符内 是
regionID string 区域id 是
description string 描述比如地址联系方式等，限制在256字符内 否
address string 影院地址，限制在256字符内 否
longitude double 经度 否
latitude double 纬度 否
phone string 影院联系方式，限制在256字符内 否

response

接口返回结果通过http状态码来标识，200代表成功，400代表请求参数错误，500代表服务器错误，401表示登录超时。

成功 200

{
  "cinemaID": "fqj39fha8ef9q83fp983q9er"
}

失败 400/500

{
  "info": "失败描述"
}

3.播放终端管理

3.1创建终端

request

POST /cms/vr_clients

Content-Type : text/plain

{
  "hallID": "fqofjp98jf9p8q3948fq3",
  "seatNumber": "02",
  "performanceLevel": 720,  
  "seatModel": "vr0109",
  "HMDModel": "hmd109",
  "MACAddr": "99:ed:9d:3b:e8:36",
  "IPAddr": "192.168.8.28",
  "description": "描述",
  "clientCode":"vrdevice",
  "actionFileVersion": "tier1",
}
名称 类型 描述 是否必须
hallID string 放映厅id 否
seatNumber string 座位号 否
performanceLevel double 终端的播放性能水平 是
seatModel string 座椅型号 否
HMDModel string 头盔型号 否
MACAddr string 座椅主机的mac地址 否
IPAddr string 座椅主机的ip地址 是
description string 描述(最长265个字) 否
actionFileVersion string 动作文件型号 是

response

成功 200

{
  "clientID": "fqj3fjp9ejfp9w8f983qf9q9p8"
}

失败 400/500

{
  "info": "失败描述"
}

request

GET /cms/vr_clients?client_id={client_id}&cinema_id={cinema_id}&hall_id={hall_id}

名称 类型 描述 是否必须
client_id string 客户端id，可多传 否
client_code string 客户端识别码(精准匹配) 否
client_code_like string 客户端识别码(模糊查询) 否
cinema_id string 影院id 否
hall_id string 播放厅id 否
qr_result string 二维码扫描结果 否
scene_guid string 微信场景号 否
start int 分页开始，默认为0 否
limit int 页面大小，默认为20 否

      "seatNumber": "02",
      "performanceLevel": 720,

座椅状态(status)枚举：INITIALIZING(初始化) / SHUT_DOWN(关机) / IDLE(空闲) / IN_VOD(点播) / PLAYING(播放) / ERROR(错误)

3.6提交终端事件

request

POST /cms/vr_clients/{scene_id}/event

{
  "event": "SCAN",
  "openId": "g928gj98es9gw45jg98rsgfp9s8u",
  "expireDate":　"1510208988793"
}
名称 类型 描述 是否必须
scene_id string 客户端sceneId 是
event string 事件类型 是
openId string 扫描用户的openId 扫描事件时必须
expireDate string 订单过期时间 订单事件时必须

本接口响应支付成功以外的设备事件, 并转交COMS执行. 支付成功由点播任务(5)接口处理.

支持的事件类型: SCAN, CREATE_TICKET. 扫描设备, 生成订单

response

成功 200

失败 400/500

{
  "info": "失败描述"
}
