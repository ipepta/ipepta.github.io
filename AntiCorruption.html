1.通用结构

1.1通用数据表结构

Content-Type : text/plain

{
  "tableSchema":{
                    "key1": "name1",
                    "key2": "name2"
                    ... 
                },
  "tableData":[
           {
             "key1": "1499",
             "key2": "2000",
             ....
           },
           {
             "key1": "399",
             "key2": "1000",
             ....
           },        
         ],
}
名称 类型 描述
tableSchema list 表头列表
tableData list 数据列表
key string 字段
name string 字段名

1.2通用参数和返回值

表格类查询均适用以下参数

名称 类型 描述 是否必填
start int 起始数据index(注意不是页码) 否, 默认0
limit int 页长度 否, 默认20

所有接口返回结果通过http状态码来标识

200代表成功，400代表请求参数错误，500代表服务器错误，401表示登录超时

类型为enum的参数/返回值请使用5.2接口进行翻译.

2.登录接口

2.1用户登录

request

POST /Auth/Login

Content-Type : text/plain

{
  "userName": "admin",
  "password": "admin"
}
名称 类型 描述 是否必须
userName string 用户名 是
password string 密码 是

response

成功 200

{
  "userName": "admin",
  "chineseName": "管理员",
  "userType": "ADMIN"
}
名称 类型 描述
userName string 用户名
chineseName string 中文名
userType string 用户类型

用户类型: ADMIN, IMPORT. 全局管理员, 导入管理员

失败 400/500

{
  "info": "失败描述"
}

3.导入接口

3.1导入数据

request

POST /Import/ImportData?dataType=BASIC&dataSource=LOCAL&importName=8月19日导入&fileName=a973fbab-e985-4363-9bc7-e29623230cfc.xlsx

名称 类型 描述 是否必须
dataType enum 文件类型 是
dataSource enum 文件来源 是
importName string 用户输入的导入名 否, 为空时后台自动生成一个
fileName string 文件名 是, 6.1接口返回

response

成功 200

{
  "importId": "fe6b0d72-d9dd-4fbd-8237-f0245513607c",
  "importName": "测试导入0827_01",
  "userChineseName": "管理员(测试)",
  "totalCount": 100,
  "insertCount": 30,
  "updateCount": 70,
  "successCount": 95,
  "failedCount": 5
}
名称 类型 描述
importId string 导入编号(不用显示)
importName string 导入名
userChineseName string 导入管理员
totalCount int 数据总数
insertCount int 新增数据
updateCount int 更新数据
successCount int 导入成功
failedCount int 导入失败

失败 400/500

{
  "info": "失败描述"
}
